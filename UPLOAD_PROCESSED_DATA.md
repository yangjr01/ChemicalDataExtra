# 上传预处理数据功能说明

## 📋 功能概述

新增"上传预处理过的数据"功能，允许用户直接上传已用LLM处理好的文献数据，无需再次调用API提取，节省时间。

## 🎯 使用场景

适用于以下情况：
- 已在外部使用LLM（如ChatGPT、Claude等）处理过PDF文献
- 已有5套提示词的处理结果（txt格式）
- 希望直接导入数据到系统进行查看和管理

## 📁 文件格式要求

上传的文件夹必须包含以下文件：

```
文件夹/
├── s12633-022-02109-4.pdf          (必需) PDF原文
├── 1预提取-材料工艺信息397.txt     (可选) 预提取结果1
├── 2材料表397.txt                  (可选) 材料结构化数据
├── 3工艺表397.txt                  (可选) 工艺结构化数据
├── 4预提取-表征信息397.txt         (可选) 预提取结果2
└── 5表征信息表397.txt              (可选) 表征结构化数据
```

**注意：**
- PDF文件**必需**，系统以此创建文献记录
- txt文件**可选**，但至少需要一个预提取结果文件
- 文件名必须以指定的前缀开头（1预提取-、2材料表-等）

## 🚀 使用步骤

### 1. 打开上传对话框
- 在文献管理页面点击右下角的 **"+" 浮动按钮**
- 或点击工具栏的 **"上传文献"** 按钮

### 2. 选择上传模式
在上传对话框中，选择 **"上传预处理过的数据"** 单选按钮

### 3. 选择文件
- 点击 **"选择文件夹（可多选）"** 按钮
- 在文件选择器中，选中包含PDF和txt文件的文件夹
- 支持多选，可同时上传多个文件

### 4. 查看已选文件
选择后，系统会显示已选文件列表：
- PDF文件显示为 **蓝色标签**
- txt文件显示为默认标签
- 系统会自动从PDF文件名提取文献标题

### 5. 填写文献信息（可选）
可以补充填写：
- **标题** - 系统自动从PDF文件名提取，可修改
- **DOI** - 文献DOI号
- **期刊** - 发表期刊
- **摘要** - 文献摘要

### 6. 点击上传
系统会：
1. 创建文献记录
2. 保存PDF文件
3. 读取所有txt文件内容
4. 创建对应的提取任务记录（extraction_tasks）

### 7. 查看结果
上传成功后：
- 文献会出现在文献列表中
- 状态显示为 **"已完成"**
- 点击 **"查看数据"** 可看到预提取的内容
- 进入提取流程时，会直接显示已预提取的结果

## 🔍 数据存储说明

上传的预处理数据存储在以下位置：

### 文献基本信息
- **表**: `chemical_articles`
- **字段**: title, doi, journal, abstract, sourceFile(PDF路径)
- **状态**: completed（已完成）

### 预提取结果
- **表**: `chemical_extraction_tasks`
- **存储内容**: 
  - `rawResponse`: txt文件的原始内容
  - `parsedData`: 解析后的数据（目前直接存储原始内容）
  - `promptName`: 对应的提示词模板名称
  - `status`: completed（已完成）

### 5个预提取模块对应关系

| 文件名前缀 | promptId | 说明 |
|-----------|---------|------|
| 1预提取-材料工艺信息 | pre_extraction_materials_processes | 材料和工艺的预提取信息 |
| 2材料表 | materials_table | 材料结构化数据表 |
| 3工艺表 | processes_table | 工艺结构化数据表 |
| 4预提取-表征信息 | pre_extraction_characterizations | 表征预提取信息 |
| 5表征信息表 | characterizations_table | 表征结构化数据表 |

## 📊 与提取流程的集成

上传预处理数据后：

1. **数据查看页面** (`/chemical/data/:id`)
   - 会显示所有预提取的内容（5个模块）
   - 在对应的标签页中查看原文

2. **提取流程页面** (`/chemical/extract/:id`)
   - 如果已存在预提取结果，会直接显示
   - 可以选择重新提取或跳过
   - 支持将预提取结果保存到结构化数据库

3. **数据汇总页面** (`/chemical/alldata`)
   - 会包含上传的预处理数据
   - 可在材料表、工艺表、表征表中查看

## ⚠️ 注意事项

1. **PDF文件必需**
   - 系统必须有PDF文件才能创建文献记录
   - txt文件是可选的，但只有PDF时无法查看预提取内容

2. **文件名匹配**
   - txt文件必须以指定的前缀开头（1预提取-、2材料表-等）
   - 不匹配的文件会被忽略

3. **文件编码**
   - 请确保txt文件使用 **UTF-8 编码**
   - 其他编码可能导致中文乱码

4. **数据格式**
   - 系统会将txt文件的原始内容直接存储
   - 不验证JSON格式，请确保内容正确

5. **重复上传**
   - 每次上传会创建新的文献记录
   - 如需更新，请删除旧记录后重新上传

## 💡 提示

- 可以只上传PDF + 部分txt文件，系统会自动识别并导入存在的预提取结果
- 上传后，预提取结果会显示在数据查看页面的对应标签页中
- 提取流程中会显示已缓存的预提取结果，可以直接使用或重新提取
