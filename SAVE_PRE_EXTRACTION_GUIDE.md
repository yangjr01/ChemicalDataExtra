# 保存预提取数据到数据库功能

## 📋 功能概述

在数据查看页面 (`/chemical/data/:articleId`) 添加了"保存到数据库"按钮，可以将预提取的材料、工艺、表征数据直接保存到数据库中。

## 🎯 使用方法

### 1. 确保有预提取数据

首先，你需要通过以下方式之一获得预提取数据：

**方式 A：上传预处理过的数据**
- 在文献管理页面上传包含预提取结果的文件夹（PDF + 5个txt文件）
- 参见 `UPLOAD_PROCESSED_DATA.md` 文档

**方式 B：在提取流程中进行提取**
- 进入提取流程页面 (`/chemical/extract/:articleId`)
- 选择对应的提示词模板进行提取
- 等待提取完成

### 2. 查看预提取结果

进入数据查看页面 (`/chemical/data/:articleId`)，你会看到：
- 页面顶部显示"预提取结果（5个模块）"卡片
- 点击"展开"按钮查看所有模块
- 已提取的模块显示绿色"已提取"标签

### 3. 保存到数据库

对于每个已提取的模块：
1. 展开该模块的折叠面板
2. 查看预提取的原始 JSON 数据
3. 点击底部的"**保存到数据库**"按钮
4. 确认保存对话框
5. 等待保存完成

### 4. 验证保存结果

保存成功后：
- 会弹出提示："保存成功！共保存 X 条记录"
- 页面会自动刷新数据
- 切换到对应的标签页（材料/工艺/表征）查看已保存的数据

## 📊 支持的模块

可以保存以下模块的预提取数据：

| 模块名称 | 保存内容 | 数据库表 |
|---------|---------|---------|
| **材料表** | 材料结构化数据 | chemical_materials |
| **工艺表** | 工艺结构化数据 | chemical_processes |
| **表征信息表** | 表征结构化数据 | chemical_characterizations |
| **预提取-材料工艺信息** | 材料和工艺预提取信息 | chemical_materials + chemical_processes |
| **预提取-表征信息** | 表征预提取信息 | chemical_characterizations |

## 🔧 技术实现

### 后端 API

**POST /api/chemical/data/:articleId/save-pre-extraction**

请求体：
```json
{
  "moduleName": "materials_table"
}
```

响应：
```json
{
  "success": true,
  "data": {
    "savedCount": 5,
    "moduleName": "materials_table"
  },
  "message": "成功保存 5 条记录到数据库"
}
```

### 数据转换逻辑

1. 从 `extraction_tasks` 表中查找对应模块的预提取结果
2. 解析原始 JSON 数据
3. 根据模块名称将数据映射到对应的数据库表
4. 保存到数据库

## 💡 注意事项

1. **数据格式**：预提取结果必须是有效的 JSON 格式
2. **重复保存**：可以多次保存同一模块，但可能会创建重复记录
3. **空数据**：如果预提取结果为空或格式不正确，会显示错误提示
4. **权限**：需要登录后才能使用此功能

## 🐛 故障排除

### 问题：点击保存按钮后提示"未找到预提取结果"

**原因**：该模块尚未进行预提取

**解决**：
- 在提取流程中先进行该模块的提取
- 或上传包含该模块预处理结果的文件夹

### 问题：保存成功但数据未显示

**原因**：页面未自动刷新

**解决**：
- 手动刷新页面
- 检查对应标签页的数据列表

### 问题：部分数据保存失败

**原因**：某些数据格式不符合要求

**解决**：
- 检查预提取结果的 JSON 格式
- 查看浏览器控制台或服务器日志获取详细信息
