# 使用 LLM 智能转换并保存预提取数据

## 📋 功能概述

此功能使用 LLM（DeepSeek）智能分析预提取的自由文本，将其转换为结构化的材料表、工艺表和表征表格式，然后保存到数据库。

## 🎯 核心逻辑

### 1. 双模式处理

系统会依次处理以下数据：

**A. 结构化数据表**（优先级最高）
- `materials_table` → 直接解析 JSON 保存到材料表
- `processes_table` → 直接解析 JSON 保存到工艺表  
- `characterizations_table` → 直接解析 JSON 保存到表征表

**B. 预提取自由文本**（使用 LLM 智能转换）
- `pre_extraction_materials_processes` → 调用 LLM 转换为结构化材料和工艺数据
- `pre_extraction_characterizations` → 调用 LLM 转换为结构化表征数据

### 2. LLM 转换流程

对于预提取的自由文本，系统会：

1. **构建提示词**：根据数据类型（材料/工艺/表征）构建专业提示词
2. **调用 LLM**：使用 DeepSeek API 进行智能分析和转换
3. **解析结果**：提取 LLM 返回的 JSON 数组
4. **保存数据**：将结构化数据保存到对应的数据库表

## 📝 提示词示例

### 材料转换提示词

```
系统角色：你是一个专业的材料科学数据整理助手...

任务：请将给定的材料信息文本转换为标准的 JSON 格式材料表。

要求：
1. 识别文本中提到的所有材料
2. 每个材料必须包含以下字段：
   - name: 材料名称（必须）
   - formula: 化学式（如果有）
   - composition: 组成成分
   - properties: 材料属性
   - type: 材料类型
   - role: 在研究中的作用
   - preparationMethod: 制备方法
```

### 工艺转换提示词

```
任务：请将给定的工艺流程信息文本转换为标准的 JSON 格式工艺表。

要求：
1. 识别文本中描述的所有工艺步骤
2. 每个工艺步骤必须包含：
   - name: 工艺名称
   - description: 工艺描述
   - sequence: 步骤序号
   - conditions: 工艺条件
   - parameters: 工艺参数
   - inputMaterials: 输入材料
   - outputMaterials: 输出材料
```

### 表征转换提示词

```
任务：请将给定的表征信息文本转换为标准的 JSON 格式表征表。

要求：
1. 识别文本中描述的所有表征测试
2. 每个表征必须包含：
   - technique: 表征技术名称
   - conditions: 测试条件
   - results: 测试结果
   - purpose: 表征目的
```

## 🔧 API 调用

**POST /api/chemical/data/:articleId/save-all-pre-extractions**

请求体：
```json
{
  "useLLM": true  // 是否使用 LLM 转换预提取文本，默认为 true
}
```

响应：
```json
{
  "success": true,
  "data": {
    "materialsCount": 10,
    "processesCount": 5,
    "characterizationsCount": 3,
    "llmConversionCount": 8,  // 由 LLM 转换的数据条数
    "totalCount": 18
  },
  "message": "成功保存 18 条记录到数据库（其中 8 条由 LLM 智能转换）"
}
```

## 💡 优势

1. **智能识别**：LLM 能够理解自由文本中的隐含信息
2. **自动补全**：自动补充缺失的字段信息
3. **标准化输出**：统一转换为标准的数据库格式
4. **双模式处理**：结构化数据直接保存，自由文本智能转换
5. **智能去重**：避免重复保存已存在的数据

## ⚠️ 注意事项

1. **API 费用**：使用 LLM 转换会产生 API 调用费用
2. **处理时间**：LLM 转换可能需要几秒钟到几十秒
3. **数据质量**：LLM 转换的准确性取决于预提取文本的质量
4. **网络依赖**：需要能够访问 DeepSeek API
