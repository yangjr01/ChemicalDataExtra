# 统一保存预提取数据功能

## 📋 功能概述

在数据查看页面 (`/chemical/data/:articleId`) 的"预提取结果（5个模块）"卡片中，添加了一个统一的"**保存到数据库**"按钮。点击后一次性将所有预提取的材料、工艺、表征数据保存到数据库。

## 🎯 使用方法

### 1. 查看预提取结果

进入数据查看页面，展开"预提取结果（5个模块）"卡片，查看所有模块的预提取数据。

### 2. 点击保存按钮

在卡片右上角点击"**保存到数据库**"按钮：
- 系统会自动识别所有已提取的模块
- 解析每个模块的 JSON 数据
- 智能去重（不会重复保存已存在的数据）

### 3. 确认保存结果

保存成功后会显示：
```
保存成功！
材料：X 条
工艺：Y 条
表征：Z 条
```

### 4. 查看保存的数据

页面会自动刷新，切换到材料/工艺/表征标签页即可查看已保存的数据。

## 🔧 技术特点

### 智能去重
- 保存前会检查数据库中是否已存在相同名称的记录
- 避免重复保存，确保数据唯一性

### 批量处理
- 一次性处理所有5个模块的数据
- 自动识别数据类型（材料/工艺/表征）
- 批量保存到对应的数据库表

### 数据格式支持
- 支持直接 JSON 数组格式
- 支持嵌套对象格式（如 `{materials: [...], processes: [...]}`）
- 自动提取 JSON 代码块中的数据

## 📊 后端 API

**POST /api/chemical/data/:articleId/save-all-pre-extractions**

响应：
```json
{
  "success": true,
  "data": {
    "materialsCount": 5,
    "processesCount": 3,
    "characterizationsCount": 2,
    "totalCount": 10
  },
  "message": "成功保存 10 条记录到数据库"
}
```

## 💡 注意事项

1. **需要预提取数据**：必须先通过上传预处理文件或提取流程获得预提取结果
2. **智能去重**：系统会自动跳过已存在的数据，避免重复
3. **批量操作**：一次性保存所有模块的数据，无需逐个点击
4. **自动刷新**：保存成功后自动刷新页面数据
